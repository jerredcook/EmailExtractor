# Toolshed audit

Findings from a deep, multi-agent code review of all 27 tools plus four cross-cutting
audits (security, architecture/testing, licensing, UX). Every `critical`/`high`
finding was re-checked by an independent adversarial verifier that re-ran the code —
**22 serious findings raised, 22 confirmed, 0 refuted** (several were honestly
downgraded a notch by the verifier). Scores are out of 10.

Checkboxes track remediation. Two items are already fixed (commit `c3b9120`).

> Generated from the review pass; treat `medium`/`low` as advisory. Reviewers ran
> the actual code in Node/headless Chrome, but a 0/22 refutation rate means the
> lower-severity items deserve a second look before acting.

## Recurring themes (the systemic view)

- **Silent wrong output is the dominant failure mode.** Security/escaping is strong
  almost everywhere; the risk is tools that show a success indicator on a wrong
  result: image-scrubber (blank on Safari), checksum (≥2GB fail), color-tools (false
  WCAG PASS), pdf-tools (blank encrypted output), media-converter (unplayable MP4 /
  empty trim), csv-json (dropped rows), json-studio (dead tree toggle).
- **Main-thread freezes / no worker.** regex-tester (ReDoS ~57s), cron-explainer
  (~10s), text-diff (27s), checksum, exif-viewer, zip-tool (bomb). One shared
  Blob-URL worker + kill-timer would fix the class.
- **Claim vs. reality gaps.** redactor pixelate 'cannot be undone', color-tools
  contrast-with-alpha, ics-event timezone note vs DST drift.
- **Google Fonts `@import`** on every page contradicts the 'nothing uploaded' pitch
  (self-host woff2).
- **No committed tests for 26/27 tools** — the shipped-and-fixed QR Reed-Solomon bug
  proves this bites. par-avion's lib/build/test triad is the house pattern to copy.

### csv-json — 8/10
_This is one of the better hand-rolled CSV tools I've reviewed: the parser passes every RFC 4180 edge case I threw at it — escaped quotes, embedded CRLF inside quoted fields, mixed line endings, lenient stray-quote recovery — with linear performance (36 MB in under a second) and a genuinely clean security posture where every user-data sink is textContent.
